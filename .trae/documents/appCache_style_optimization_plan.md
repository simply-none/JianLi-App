# 应用缓存页面样式优化计划

## 1. 问题分析

### 当前布局问题
- 使用传统 `el-form` 表单布局，样式简陋
- 数据展示使用简单的嵌套 div，层次不清晰
- 对象/数组展示方式单调，缺乏交互性
- 与设置页面其他优化后的组件风格不统一
- "数据还原"和"数据备份"按钮样式普通

### 需要优化的内容
1. **布局风格**：摒弃表单布局，改用卡片式布局
2. **数据展示**：优化对象/数组的展示方式，参考主流网站的 JSON 展示风格
3. **操作按钮**：使用更现代化的按钮样式

---

## 2. 优化方案

### 2.1 布局设计
- 使用卡片式布局，四个功能区域各一个卡片
  - 数据还原卡片（上传 + 待还原数据预览）
  - 数据备份卡片
  - 当前缓存数据卡片
- 每个卡片有独立的标题和内容区域
- 使用与其他设置页面一致的设计风格

### 2.2 数据展示优化
- 创建一个通用的 JSON 树组件 `JsonTree.vue`
- 支持对象和数组的递归展开/折叠
- 使用彩色标签区分不同类型（字符串、数字、布尔、null）
- 支持点击展开/折叠节点
- 支持复制值功能

### 2.3 技术方案
- 使用自定义组件实现 JSON 树展示（不依赖额外插件）
- 保持轻量级，只展示必要信息
- 提供清晰的视觉层次
