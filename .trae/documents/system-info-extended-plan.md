# 系统信息页面 - 冷门信息展示优化计划

## 一、需求分析

**用户需求**：在系统信息页面中添加"冷门/不常见"的系统信息，默认不显示，通过"展示所有"按钮切换显示。

**技术背景**：
- 当前使用 `systeminformation` 库获取系统信息
- 该库提供了 50+ API，目前只使用了 6 个常用接口
- 页面已使用卡片式布局，支持主题切换

## 二、冷门信息分类

根据 `systeminformation` 库 API，以下信息属于不常用但有价值的类别：

| 类别 | API 方法 | 说明 |
|------|---------|------|
| BIOS 信息 | `si.bios()` | 主板 BIOS 版本、厂商、序列号 |
| 主板信息 | `si.baseboard()` | 主板型号、厂商、序列号 |
| 电池信息 | `si.battery()` | 电池状态、电量、健康度 |
| 音频设备 | `si.audio()` | 音频控制器和设备列表 |
| 蓝牙设备 | `si.bluetooth()` | 蓝牙适配器和连接设备 |
| USB 设备 | `si.usb()` | USB 控制器和连接设备 |
| 打印机 | `si.printer()` | 打印机设备列表 |
| Docker | `si.dockerInfo()` | Docker 容器信息 |
| 软件版本 | `si.versions()` | Node、NPM、Git 等版本 |
| 用户信息 | `si.users()` | 当前在线用户 |

## 三、修改方案

### 1. 修改 Worker 文件 (`public/worker/systemInfo.cjs`)

**新增功能**：
- 添加 `getExtendedInfo()` 函数，并行获取所有冷门信息
- 添加 `extended` 消息类型处理

**关键点**：
- 使用 `Promise.allSettled` 确保部分接口失败不影响整体
- 对每个接口进行错误捕获，返回空对象而非抛出异常

### 2. 修改页面组件 (`src/views/systemInfo/systemInfo.vue`)

**新增功能**：
- 在页面头部添加"展示所有"按钮
- 添加 `showExtended` 状态控制
- 添加扩展信息区域（默认隐藏）
- 使用折叠面板展示分类信息

**布局设计**：
- 扩展信息使用 Accordion（折叠面板）组件
- 每个分类独立一个折叠项
- 采用与现有卡片一致的样式风格

**信息分类展示**：
- **硬件详情**：BIOS、主板、电池
- **外设设备**：音频、蓝牙、USB、打印机
- **软件环境**：Docker、软件版本、用户信息

## 四、性能考虑

1. **延迟加载**：扩展信息仅在用户点击"展示所有"按钮时才获取
2. **并行获取**：使用 `Promise.allSettled` 并行请求所有扩展信息
3. **错误隔离**：单个接口失败不影响其他信息展示
4. **按需展示**：获取后缓存数据，切换显示时无需重新请求

## 六、风险评估

| 风险 | 等级 | 应对措施 |
|------|------|---------|
| 部分 API 在 Windows 上不可用 | 中 | 使用 `Promise.allSettled` 容错，只显示可用信息 |
| 扩展信息获取较慢 | 中 | 延迟加载，用户点击时才获取 |
| 数据量大导致页面卡顿 | 低 | 使用折叠面板，按需渲染 |
