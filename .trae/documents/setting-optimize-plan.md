# 设置页面样式优化计划 - 卡片式内部布局

## 一、需求分析

**用户需求**：保持从上往下的整体布局，但优化每个设置项内部的展示效果，使用卡片和新颖的设计。

**当前问题**：
1. 使用简单的 el-form 标签布局，样式简陋
2. 背景色硬编码为白色，不支持主题切换
3. 没有使用项目定义的 CSS 变量
4. 设置项内部缺乏视觉层次和创意

### 2. 番茄钟状态卡片

**设计特点**：
- 横向展示状态信息，使用彩色标签
- 工作中：绿色背景 + 脉冲动画点
- 休息中：红色背景 + 脉冲动画点
- 下次工作/休息时间紧凑排列

```
┌──────────────────────────────────────────────────────────────┐
│  <div class="status-bar">                                    │
│    <span class="status-tag work">● 工作中</span>             │
│    <span class="status-time">下次工作: 14:30</span>          │
│    <span class="status-time">下次休息: 14:55</span>          │
│  </div>                                                      │
└──────────────────────────────────────────────────────────────┘
```

### 6. 快捷操作按钮组

**设计特点**：
- 使用 2x2 网格布局
- 按钮带有图标和文字
- 不同功能使用不同颜色区分

```

└─────────────────────┘ └─────────────────────┘
```

### 7. 主题兼容性

**使用项目 CSS 变量**：
- `--bg-base` - 页面背景
- `--bg-card` - 卡片背景
- `--text-primary` - 主文字
- `--text-secondary` - 次要文字
- `--color-primary` - 主题色
- `--border-subtle` - 边框
- `--radius-card` - 卡片圆角
- `--shadow-card` - 卡片阴影

### 8. 性能优化

**优化方案**：
- 使用 `onMounted` + `nextTick` 加载字体数据
- 将字体加载改为异步加载
- 简化 watch 逻辑，减少不必要的深拷贝
