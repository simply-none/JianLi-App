# 笔记本页面优化计划

## 一、现状分析

### 1.1 当前页面问题
| 问题类别 | 具体问题 | 影响 |
|---------|---------|------|
| 布局结构 | 使用简单的 el-form 平铺布局，缺乏层次 | 用户体验差，信息分区不清晰 |
| 视觉设计 | 使用基础的 el-table 展示笔记，样式死板 | 界面老旧，缺乏现代感 |
| 交互体验 | 日历和列表布局拥挤，操作不便 | 用户难以快速找到和管理笔记 |
| 样式风格 | 硬编码颜色值，未接入主题系统 | 不支持深色主题切换 |
| 编辑器布局 | 编辑器区域没有合理的布局规划 | 编辑体验不佳 |

### 1.2 参考主流笔记应用设计
- **Notion**：侧边栏导航 + 主编辑区，简洁现代
- **印象笔记**：日历视图 + 卡片式笔记列表
- **Flomo**：时间轴式笔记展示，极简设计

## 二、优化方案

### 2.1 布局结构优化

#### 方案：三栏式布局
```
┌─────────────────────────────────────────────────────────────┐
│  📝 笔记本                     [搜索] [新建] [设置]          │
├─────────────┬───────────────────┬───────────────────────────┤
│             │                   │                           │
│  日历侧边栏 │   笔记列表        │       编辑器区域           │
│             │                   │                           │
│  2024年1月  │  • 笔记1摘要      │                           │
│  ┌───────┐  │    09:30 更新     │                           │
│  │ 1  2  │  │                   │                           │
│  │ 3  4• │  │  • 笔记2摘要      │                           │
│  │ 5  6  │  │    昨天 14:20     │                           │
│  │ ...   │  │                   │                           │
│  └───────┘  │  • 笔记3摘要      │                           │
│             │    周一 10:00     │                           │
│             │                   │                           │
│             └───────────────────┴───────────────────────────┘
│             │                   │                           │
└─────────────┴───────────────────┴───────────────────────────┘
```

#### 布局特点
- **左侧日历栏**：垂直日历，显示有笔记的日期标记
- **中间列表栏**：卡片式笔记列表，按时间倒序排列
- **右侧编辑栏**：全屏编辑器，支持新建和编辑

### 2.2 设计要点

#### 2.2.1 日历侧边栏
- 简洁的月历视图
- 有笔记的日期显示圆点标记
- 当前日期高亮
- 点击日期过滤笔记

#### 2.2.2 笔记列表
- 卡片式设计，hover 效果
- 显示摘要、时间、标签
- 支持搜索和筛选
- 平滑滚动

#### 2.2.3 编辑器区域
- 顶部工具栏（新建、保存、清空）
- 编辑器占据剩余空间
- 标题显示当前状态

### 2.3 样式优化

#### 2.3.1 CSS变量应用
```scss
background: var(--bg-card);
border: 1px solid var(--border-subtle);
border-radius: var(--radius-card);
box-shadow: var(--shadow-card);
color: var(--text-primary);
```

#### 2.3.2 交互效果
- 卡片 hover 上浮 + 边框高亮
- 日历日期 hover 变色
- 平滑过渡动画

### 步骤3: 优化 umoEditor.vue

## 五、风险与注意事项

| 风险点 | 应对措施 |
|--------|---------|
| 功能逻辑破坏 | 保留原有 script 逻辑，仅修改 template 和 style |
| 主题兼容性 | 使用 CSS 变量确保适配所有主题 |
| 响应式问题 | 使用 flex 布局确保不同屏幕尺寸正常显示 |
| 编辑器高度问题 | 设置 flex:1 确保编辑器占满剩余空间 |
