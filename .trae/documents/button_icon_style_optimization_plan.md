# 按钮图标样式优化计划

## 1. 问题分析

### 当前问题
- 按钮设置了 `type="primary"`、`type="warning"`、`type="danger"` 等属性
- Element Plus 的 type 属性会给按钮添加背景色，导致图标颜色被背景色覆盖或不够明显
- CSS 中设置的 `color` 可能被 Element Plus 默认样式优先级覆盖
- 小尺寸圆形按钮中，图标与背景对比度不够

### 具体问题点
1. **pomodoro.vue** 中的操作按钮（编辑/开始/停止/删除）背景色覆盖了图标颜色
2. **pomodoro.vue** 中的提醒类型操作按钮同样存在此问题
3. 按钮颜色语义不明确，用户难以快速识别功能

---

## 2. 优化方案

### 2.1 按钮样式优化
- 移除按钮的 `type` 属性，使用纯图标按钮
- 使用 `text` 类型或完全自定义样式
- 添加悬停时的背景色变化（浅色背景 + 彩色图标）
- 确保图标颜色在各种状态下都清晰可见

### 2.2 颜色方案
| 功能 | 图标颜色 | 悬停背景色 |
|------|---------|-----------|
| 编辑 | #409eff (蓝色) | rgba(64, 158, 255, 0.1) |
| 开始/播放 | #67c23a (绿色) | rgba(103, 194, 58, 0.1) |
| 停止/暂停 | #e6a23c (橙色) | rgba(230, 162, 60, 0.1) |
| 删除 | #f56c6c (红色) | rgba(245, 108, 108, 0.1) |

### 2.3 交互效果
- 默认状态：透明背景 + 彩色图标
- 悬停状态：浅色背景 + 彩色图标 + 轻微放大
- 点击状态：背景色加深 + 轻微缩小
