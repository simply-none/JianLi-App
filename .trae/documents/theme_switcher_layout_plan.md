# 主题切换器排版优化计划

## 一、当前问题分析

**文件位置**: `src/layout/index.vue`

当前主题切换器将所有主题（10个）横向排列在一行，导致：
- 视觉拥挤，间距过小
- 鼠标悬停时容易误触相邻主题
- 侧边栏空间有限（约210px），无法良好展示

## 二、优化方案

### 方案设计

**主区域显示**:
- 当前选中的主题（始终显示）
- 1个推荐/常用主题（可配置或按使用频率）
- 更多按钮（"⋮" 三点样式）

**展开面板**:
- 点击更多按钮时，在侧边栏下方弹出下拉面板
- 显示剩余的所有主题
- 支持滚动（如果主题数量继续增长）
- 点击外部区域关闭面板

### UI 交互流程

```
┌─────────────────────┐
│  主题  [●] [●] [⋮]  │  ← 主区域：当前主题 + 1个 + 更多
└─────────────────────┘
         ↓ 点击 [⋮]
┌─────────────────────┐
│  主题  [●] [●] [▼]  │
│ ┌─────────────────┐ │
│ │ [●] 浅色        │ │
│ │ [●] 深色        │ │
│ │ [●] One Dark    │ │  ← 下拉面板
│ │ [●] Dracula     │ │
│ │ ...             │ │
│ └─────────────────┘ │
└─────────────────────┘
```

## 三、实施步骤

### 步骤1: 修改主题切换器模板结构
- 将 `theme-options` 改为固定显示2个主题 + 更多按钮
- 添加下拉面板组件（使用 Element Plus 的 `el-dropdown` 或自定义）
- 绑定点击事件和面板显示状态

### 步骤2: 更新样式
- 调整主题点的尺寸和间距
- 添加下拉面板样式（圆角、阴影、背景色）
- 添加过渡动画效果

### 步骤3: 逻辑优化
- 计算显示的主题（当前主题优先 + 1个其他主题）
- 计算展开面板中显示的主题（排除已显示的）
- 添加点击外部关闭面板的逻辑

## 四、文件修改清单

| 文件 | 修改内容 |
|------|---------|
| `src/layout/index.vue` | 主题切换器模板、样式、逻辑 |

## 五、潜在风险与注意事项

1. **响应式适配**: 确保在不同主题下下拉面板样式一致
2. **动画流畅度**: 面板展开/收起需要平滑过渡
3. **点击穿透**: 确保点击面板外部能正确关闭
4. **主题数量变化**: 如果未来新增主题，下拉面板需支持滚动
