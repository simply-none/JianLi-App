# 番茄钟小窗口皮肤与排版主题设计方案

## 一、需求分析

### 1.1 问题描述
当前番茄钟小窗口使用固定的半透明背景，在某些背景下会融合进其他应用，导致难以辨认。

### 1.2 功能需求
1. **多皮肤支持**：直接使用 `style.scss` 中已有的主题配色（dark, nord, dracula 等），不需要重新设计
2. **手动切换**：用户点击窗口右上角区域切换皮肤/颜色
3. **多排版主题**：将当前排版封装为"默认"主题，再增加几种主流番茄钟布局

### 1.3 技术约束
- 窗口尺寸可能很小（如 108x81），所有内容必须适配
- 使用响应式单位（clamp(), vmin, em）
- 保留现有功能：状态指示、倒计时、进度条、鼠标穿透

## 二、设计方案

### 2.1 皮肤系统设计

#### 2.1.1 皮肤类型

直接使用 `style.scss` 中已有的主题名称，按顺序循环切换：

| 皮肤名称 | 对应主题 | 描述 |
|---------|---------|------|
| `default` | :root | 默认浅色主题 |
| `dark` | dark (Tokyo Night) | 深色霓虹风格 |
| `midnight` | midnight (GitHub Dimmed) | 午夜暗色 |
| `nord` | nord (北极) | 冷色调暗色 |
| `one-dark` | one-dark (Atom) | Atom 经典暗色 |
| `dracula` | dracula | 高对比度暗色 |
| `github-dark` | github-dark | GitHub 官方暗色 |
| `tokyo-night` | tokyo-night | 东京夜霓虹 |
| `solarized` | solarized | 科学配色暗色 |
| `gruvbox` | gruvbox | 温暖复古暗色 |
| `catppuccin` | catppuccin | 浅色奶咖 |
| `catppuccin-mocha` | catppuccin-mocha | 深色奶咖 |
| `ayu-dark` | ayu-dark | Ayu 暗色 |
| `ayu-mirage` | ayu-mirage | Ayu 幻彩 |

#### 2.1.2 皮肤切换逻辑

通过动态设置 `<html>` 的 `data-theme` 属性来切换：

```typescript
const themes = [
  'default', 'dark', 'midnight', 'nord', 'one-dark', 
  'dracula', 'github-dark', 'tokyo-night', 'solarized', 
  'gruvbox', 'catppuccin', 'catppuccin-mocha', 'ayu-dark', 'ayu-mirage'
]

const cycleTheme = () => {
  const currentTheme = document.documentElement.getAttribute('data-theme') || 'default'
  const idx = themes.indexOf(currentTheme)
  const nextTheme = themes[(idx + 1) % themes.length]
  
  if (nextTheme === 'default') {
    document.documentElement.removeAttribute('data-theme')
  } else {
    document.documentElement.setAttribute('data-theme', nextTheme)
  }
  
  saveTheme(nextTheme)
}
```

#### 2.1.3 交互区域

在窗口右上角添加一个可点击区域（不影响鼠标穿透功能）：

```
├─────────────────────────┤
│        25:00            │
├─────────────────────────┤
│ ████████████░░░░ 75%    │
└─────────────────────────┘
```

### 2.2 排版主题设计

#### 2.2.1 主题一：默认（当前布局）

```
┌─────────────────────────┐
│ ● 工作中    距离休息    │ ← 顶部状态栏
├─────────────────────────┤
│        25:00            │ ← 中间倒计时
├─────────────────────────┤
│ ████████████░░░░ 75%    │ ← 底部进度条
└─────────────────────────┘
```

#### 2.2.2 主题二：简约（极简风格）

```
┌─────────────────────────┐
│      25:00              │ ← 大号倒计时居中
│        ● 工作中         │ ← 状态指示在下方
└─────────────────────────┘
```

**特点**：无进度条，只有时间和状态，最简洁

#### 2.2.3 主题三：圆形（环形进度）

```
┌─────────────────────────┐
│    ┌──────────┐         │
│    │   25:00  │         │ ← 环形进度条 + 时间
│    │   ●工作  │         │
│    └──────────┘         │
└─────────────────────────┘
```

**特点**：圆形进度条环绕时间，视觉冲击力强

#### 2.2.4 主题四：紧凑（横排布局）

```
┌─────────────────────────┐
│ ● 工作中  25:00  ████░░ │ ← 所有内容一行显示
└─────────────────────────┘
```

**特点**：横向布局，适合窄窗口

#### 2.2.5 主题五：经典（传统番茄钟）

```
┌─────────────────────────┐
│       🍅 番茄钟         │ ← 标题
├─────────────────────────┤
│        25:00            │ ← 大号时间
│       工作中 75%        │ ← 状态+进度
└─────────────────────────┘
```

**特点**：带标题，温馨风格

## 三、交互设计

### 3.1 皮肤切换

**触发方式**：点击窗口右上角区域（约 20% 宽度）

**切换逻辑**：循环切换 themes 数组中的主题

**视觉反馈**：切换时有 0.3s 淡入淡出动画

### 3.2 排版切换

**触发方式**：双击窗口任意区域（除左右鼠标穿透区和右上角切换区）

**切换逻辑**：
```
default → simple → circle → compact → classic → default
```

### 3.3 状态持久化

- 当前皮肤和排版主题存储到本地数据库
- 下次打开小窗口时自动加载上次设置

## 五、风险与注意事项

### 5.1 风险
1. **窗口尺寸过小**：某些排版在极小窗口下可能显示不全
2. **主题切换冲突**：切换主题会影响整个应用，不仅仅是小窗口
3. **状态同步问题**：主题切换后需要确保状态颜色正确

### 5.2 解决方案
1. 使用 `clamp()` 和 `vmin` 确保所有元素自适应
2. 仅在小窗口中切换主题，主应用不受影响（通过独立的 CSS 变量作用域）
3. 使用 CSS 变量确保颜色自动适配当前主题
