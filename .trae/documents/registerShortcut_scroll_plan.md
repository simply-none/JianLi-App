# 快捷键注册页面滚动方式修改计划

## 需求分析

将快捷键注册页面的滚动方式从**整个页面滚动**改为**快捷键卡片区域滚动**，保持头部固定。

## 当前问题分析

当前页面结构：
```
.shortcut-page (min-height: 100vh)
  ├── .page-header (渐变背景头部)
  └── .page-content (包含网格和提示)
        ├── .shortcuts-grid (快捷键卡片网格)
        └── .tips-section (使用说明)
```

问题：整个 `.shortcut-page` 可以滚动，导致头部也会跟着滚动。

## 修改方案

### 修改步骤

1. **修改 `.shortcut-page`**：
   - 将 `min-height: 100vh` 改为 `height: 100vh`
   - 添加 `overflow: hidden` 防止页面整体滚动

2. **修改 `.page-content`**：
   - 添加 `overflow-y: auto` 实现内容区域滚动
   - 设置 `flex: 1` 让内容区域占满剩余空间

3. **调整响应式布局**：
   - 在移动端保持相同的滚动行为

## 风险与注意事项

1. **内容裁剪**：确保内容区域有足够的滚动空间
2. **响应式适配**：移动端也需要保持头部固定、内容滚动
3. **滚动条样式**：确保滚动条显示正常

## 验证方案

1. 打开快捷键注册页面，调整窗口高度
2. 验证头部保持固定不动
3. 验证快捷键卡片区域可以正常滚动
4. 验证底部使用说明也在滚动区域内
