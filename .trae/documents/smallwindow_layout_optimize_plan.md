# 小窗口模式布局优化计划

## 一、仓库调研结论

### 当前代码结构

**[smallWindow.vue](file:///C:/cod/electron-vite-vue/src/views/smallWindow.vue)**
- 使用简单的 Flex 布局，展示工作/休息状态
- 包含三个信息：状态标签、下次时间、倒计时
- 设置按钮使用 SVG 图片，位于右上角
- 支持窗口拖拽（-webkit-app-region: drag）

### 当前问题分析

| 问题 | 影响 |
|------|------|
| 信息层次不清晰 | 用户难以快速获取关键信息 |
| 缺少视觉区分 | 工作/休息状态没有明显的视觉差异 |
| 倒计时不够突出 | 核心信息不够醒目 |
| 布局过于基础 | 缺乏现代感和美观度 |
| 设置按钮交互反馈不足 | 仅有 hover 效果，缺少点击反馈 |

## 二、优化方案

### 1. 新布局设计

**整体布局结构：**
```
┌──────────────────────────────┐
│  ┌──┐  工作中                 │
│  │⏱│  下次休息: 14:30        │
│  └──┘  ○○○○○○○○○○○○○○○○○○○○○  │
│        00:23:45              │
└─────────────┬────────────────┘
              │ 设置按钮
              ▼
```

**布局特点：**
- 状态图标 + 状态文字组合，清晰区分工作/休息
- 进度条可视化展示当前阶段进度
- 倒计时放大显示，更加醒目
- 设置按钮融入整体布局，更加协调

### 2. 视觉优化

**工作状态（绿色主题）：**
- 状态图标：🌿 或 ▶️
- 背景渐变：浅绿色渐变
- 进度条颜色：绿色系

**休息状态（蓝色主题）：**
- 状态图标：☕ 或 ⏸️
- 背景渐变：浅蓝色渐变
- 进度条颜色：蓝色系

### 3. 新增功能

- **进度条**：根据当前时间与目标时间的比例，显示可视化进度
- **状态动画**：状态切换时添加平滑过渡动画
- **更好的交互反馈**：按钮点击、hover 效果优化

## 四、技术实现细节

### 1. 进度计算

```javascript
const progress = computed(() => {
  const now = new Date().getTime()
  const start = new Date(curStatus.value === 'work' ? startWorkTime.value : closeWorkTime.value).getTime()
  const end = new Date(curStatus.value === 'work' ? nextRestTime.value : nextWorkTime.value).getTime()
  const total = end - start
  const current = now - start
  return Math.max(0, Math.min(100, (current / total) * 100))
})
```

### 2. 状态主题

```javascript
const statusTheme = computed(() => {
```

### 3. 样式设计要点

- 使用 CSS 渐变背景区分状态
- 进度条使用圆角和阴影效果
- 字体大小层次分明（状态标题 > 倒计时 > 其他信息）

## 五、潜在依赖与风险

### 依赖
- Element Plus 图标（已存在）
- Vue 3 Composition API（已存在）

### 风险点

| 风险 | 影响 | 解决方案 |
|------|------|----------|
| 时间计算精度 | 进度条可能出现负数或超过100% | 添加边界检查 |
| 动画性能 | 小窗口频繁更新可能卡顿 | 使用 requestAnimationFrame |
| 主题切换 | 状态切换时颜色变化可能突兀 | 添加过渡动画 |
