# 安全防护页面优化计划

## 一、现状分析

### 1.1 当前页面问题
| 问题类别 | 具体问题 | 影响 |
|---------|---------|------|
| 布局结构 | 使用简单的el-form平铺布局，缺乏层次 | 用户体验差，信息分区不清晰 |
| 样式风格 | 使用硬编码颜色值，未接入主题系统 | 不支持深色主题切换，视觉一致性差 |
| 视觉设计 | 缺乏图标、阴影、圆角等现代UI元素 | 界面老旧，交互反馈弱 |
| 密码强度 | 使用单一进度条展示，缺乏颜色和文字提示 | 用户难以直观判断密码强度 |
| 密保问题 | 编辑区域布局杂乱，缺少视觉引导 | 用户操作流程不清晰 |

### 1.2 项目设计规范参考
从 `setting/index.vue` 和 `homeMode/index.vue` 提取的设计规范：
- **卡片布局**: 使用 `--bg-card` 背景色 + `--border-subtle` 边框 + `--radius-card` 圆角 + `--shadow-card` 阴影
- **标题样式**: 主标题带图标 + 底部2px主题色边框
- **交互反馈**: hover效果、过渡动画、状态指示器
- **响应式**: 使用 grid/flex 布局

### 2.2 组件拆分与重构

#### 2.2.1 密码设置区域
- **原密码展示**: 使用状态标签替代纯文本展示
- **密码校验**: 输入框 + 校验按钮 + 校验失败提示
- **新密码**: 输入框 + 实时强度指示器（颜色渐变 + 文字描述）
- **确认密码**: 输入框 + 密码匹配提示

#### 2.2.2 密保问题区域
- **折叠面板**: 使用 el-collapse 或自定义展开/收起
- **问题卡片**: 每个问题独立卡片展示，支持编辑/删除
- **新增按钮**: 添加图标，hover效果

### 2.3 样式优化

#### 2.3.1 CSS变量应用
```scss
// 替代硬编码颜色
background: var(--bg-card);
border: 1px solid var(--border-subtle);
border-radius: var(--radius-card);
box-shadow: var(--shadow-card);
color: var(--text-primary);
```

#### 2.3.2 密码强度指示器优化
| 强度等级 | 颜色 | 描述 |
|---------|------|------|
| 0 | #f56c6c | 弱 |
| 1-2 | #e6a23c | 一般 |
| 3-4 | #67c23a | 强 |
| 5 | #409eff | 非常强 |

#### 2.3.3 交互效果
- 卡片 hover 上浮 + 边框高亮
- 按钮点击缩放反馈
- 输入框聚焦边框高亮

## 四、实施步骤

### 步骤1: 重构模板结构
- 移除旧的 el-form 布局
- 添加卡片容器和分区标题
- 重构密码设置区域
- 重构密保问题区域

### 步骤2: 优化样式系统
- 替换硬编码颜色为CSS变量
- 添加卡片阴影和圆角
- 优化密码强度指示器样式
- 添加交互动画效果

### 步骤3: 增强视觉元素
- 添加图标（使用 Element Plus Icons）
- 添加状态标签和提示信息
- 优化密码强度文字描述

### 步骤4: 测试验证
- 验证功能完整性（密码校验、重置、密保验证）
- 验证主题切换兼容性（深色/浅色）
- 验证响应式布局

## 五、风险与注意事项

| 风险点 | 应对措施 |
|--------|---------|
| 功能逻辑破坏 | 保留原有script逻辑，仅修改template和style |
| 主题兼容性 | 使用CSS变量确保适配所有主题 |
| 响应式问题 | 使用flex/grid布局确保不同屏幕尺寸正常显示 |
