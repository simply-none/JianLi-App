# JsonTree 组件排版优化计划

## 1. 需求分析

### 当前问题
- 字段排列杂乱，key 和 value 在同一行但布局不够清晰
- 对象/数组展开方式不直观
- 数组索引未正确显示

### 修改要求
1. **布局调整**：每行左边展示 key，右边展示 value
2. **展开方式**：当 value 为对象时，点击展开后在当前行下方显示子内容，而非右侧展开
3. **数组处理**：数组的 key 显示为索引（0, 1, 2...）
4. **精简显示**：移除无关字段（如类型标签），只显示 key 和 value

---

## 2. 优化方案

### 2.1 布局设计
- 使用 flex 布局，左边 key，右边 value
- key 使用固定宽度或占比
- value 占剩余空间
- 对象/数组点击后在当前行下方展开子节点

### 2.2 展开方式
- 点击整个节点行（或展开按钮）时，在当前行下方展开子内容
- 子内容缩进显示，形成清晰的层次结构
- 展开图标放在 key 前面

### 2.3 数组处理
- 数组索引作为 key 显示（0, 1, 2...）
- 数组项如果是对象，继续递归展示

### 2.4 精简显示
- 移除类型标签（Object、Array[length]）
- 只显示 key 和 value
- value 根据类型显示不同颜色
