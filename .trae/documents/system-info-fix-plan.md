# 系统信息页面 - 修复数据获取和样式优化计划

## 一、问题分析

### 1. API 方法名错误

通过检查 systeminformation 5.27.11 版本的实际可用方法，发现以下问题：

| 错误方法名 | 正确方法名 |
|-----------|-----------|
| `si.bluetooth()` | `si.bluetoothDevices()` |

**影响**：调用不存在的方法导致 `TypeError: si.bluetooth is not a function`，使整个 `Promise.allSettled` 失败，返回空数据。

### 2. 样式排版问题

当前扩展信息区域使用折叠面板（Collapse）布局，用户反馈：
- 样式简陋，与主页面卡片风格不一致
- 信息展示不够直观
- 需要改为卡片式布局

## 二、修复方案

### 1. 修改 Worker 文件 (`public/worker/systemInfo.cjs`)

**修复内容**：
- 将 `si.bluetooth()` 改为 `si.bluetoothDevices()`
- 添加方法存在性检查，避免调用不存在的方法
- 优化错误处理，单个方法失败不影响其他数据

### 2. 修改页面组件 (`src/views/systemInfo/systemInfo.vue`)

**样式优化**：
- 将扩展信息区域从折叠面板改为卡片式网格布局
- 保持与主页面一致的卡片风格（圆角、阴影、边框）
- 使用网格布局排列各类扩展信息卡片
- 每个分类独立一个卡片组

## 四、风险评估

| 风险 | 等级 | 应对措施 |
|------|------|---------|
| 某些 API 在 Windows 上不可用 | 中 | 使用方法存在性检查 + Promise.allSettled |
| 数据量大导致页面卡顿 | 低 | 使用卡片网格布局，按需渲染 |
