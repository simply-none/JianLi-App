# 新增/编辑功能迁移到 Dialog 计划

## 1. 问题分析

### 当前问题
- 新增提醒表单直接显示在页面上，占用空间较大
- 新增提醒类型表单直接显示在页面上，布局冗长
- 编辑功能是直接回填到新增表单中，用户体验不直观
- 页面内容过多，视觉层次不够清晰

### 需要改造的功能
1. **pomodoro.vue** - 新增提醒
2. **pomodoro.vue** - 编辑提醒
3. **pomodoro.vue** - 新增提醒类型
4. **pomodoro.vue** - 编辑提醒类型
5. **cacheSet.vue** - 目录选择（当前已够用，无需改造）

---

## 2. 优化方案

### 2.1 pomodoro.vue 改造

#### 提醒列表区域
- 在"当前提醒"标题右侧添加"新增提醒"按钮
- 点击按钮弹出 Dialog 进行新增操作
- 点击编辑按钮弹出 Dialog 进行编辑操作
- 移除页面上的"新增提醒"表单区域

#### 提醒类型区域
- 在"提醒类型"标题右侧添加"新增类型"按钮
- 点击按钮弹出 Dialog 进行新增操作
- 点击编辑按钮弹出 Dialog 进行编辑操作
- 移除页面上的"新增提醒类型"表单区域

### 2.2 Dialog 设计
- 使用 Element Plus 的 `el-dialog` 组件
- 表单在 Dialog 内部垂直排列
- 底部放置"确定"和"取消"按钮
- Dialog 宽度：500px 左右
- 编辑时自动回填数据，新增时清空数据
- 关闭 Dialog 时重置表单

## 5. 风险与注意事项

### 风险点
1. **表单数据回填**：编辑时需要正确回填数据
2. **表单重置**：关闭 Dialog 后需要重置表单状态
3. **编辑和新增区分**：需要正确区分是新增还是编辑模式
4. **图标上传**：提醒类型 Dialog 中的图标上传功能需要正常工作

### 验证步骤
1. 确认新增提醒 Dialog 打开正常，表单可提交
2. 确认编辑提醒 Dialog 打开正常，数据正确回填
3. 确认新增提醒类型 Dialog 打开正常，图标上传功能正常
4. 确认编辑提醒类型 Dialog 打开正常，数据正确回填
5. 确认 Dialog 关闭后表单数据重置
6. 确认所有原有功能保持不变
