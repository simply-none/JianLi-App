# 快捷键注册页面样式优化计划

## 一、仓库调研结论

### 当前代码结构

**1. 主页面 [index.vue](file:///C:/cod/electron-vite-vue/src/views/registerShortcut/index.vue)**
- 使用 Element Plus 的 `el-form` 和 `el-form-item` 传统表单布局
- 包含 9 个快捷键配置项（显示应用、打开首屏、打开记事本等）
- 每个配置项包含名称、快捷键选择组件、注册按钮
- 功能逻辑完整：数据加载、合并、注册、验证

**2. 子组件 [shortcut.vue](file:///C:/cod/electron-vite-vue/src/views/registerShortcut/shortcut.vue)**
- 支持最多 3 个键组合（Ctrl+Shift+A 等）
- 当前仅支持下拉选择方式
- 有重复快捷键检测逻辑
- 样式较为基础

### 当前问题分析

| 问题 | 影响 |
|------|------|
| 传统表单布局不够直观 | 用户操作不够便捷 |
| 仅支持下拉选择 | 键盘输入更快速但不支持 |
| 样式基础，缺乏视觉层次感 | 用户体验不佳 |
| 快捷键选择区域较小 | 选择不够方便 |

## 二、优化方案

### 1. 页面布局优化（大胆变化）

**新布局设计：**
- 采用**卡片网格布局**替代传统表单布局
- 每个快捷键配置项独立为一个卡片
- 卡片内部包含：功能图标、功能名称、快捷键配置区、注册按钮
- 整体视觉层次分明，操作更直观

### 2. 快捷键选择组件优化

**核心改进：**
- **双模式支持**：同时支持键盘输入和下拉选择
- **键盘输入显示**：点击输入区域后，按下键盘按键自动识别并显示
- **下拉选择保留**：保留原有下拉选择功能，方便用户精确选择
- **快捷键视觉化**：使用类似键盘按键的样式展示已选快捷键

**快捷键输入区域设计：**
```
┌──────────────────────────────────────────────┐
│  [ Ctrl ]  [ Shift ]  [ A ]  [ ⌫ ]          │
│   点击输入或下拉选择                          │
└──────────────────────────────────────────────┘
```

**功能特点：**
1. 每个按键位置支持点击激活输入模式
2. 激活后监听键盘事件，自动识别并填充
3. 支持删除键清除当前输入
4. 支持下拉选择作为备选方案

## 五、潜在依赖与风险

### 依赖
- Element Plus（已存在）
- Vue 3 Composition API（已存在）

### 风险点

| 风险 | 影响 | 解决方案 |
|------|------|----------|
| 键盘事件冲突 | 可能与系统快捷键冲突 | 使用 `event.preventDefault()` 阻止默认行为 |
| 按键识别兼容性 | 不同操作系统按键名称可能不同 | 添加按键名称映射表 |
| 焦点管理 | 多个输入区域需要正确管理焦点 | 使用 ref 跟踪当前激活的输入框 |
